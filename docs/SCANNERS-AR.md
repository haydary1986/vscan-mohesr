## منهجية التقييم

### مقياس الدرجات

جميع الفحوصات الفردية تُقيَّم على مقياس **0–1000** نقطة:

| النطاق | الدرجة | الحالة | التفسير |
|--------|--------|--------|---------|
| 900–1000 | A | ناجح (pass) | ممتاز — يتبع أفضل الممارسات |
| 750–899 | B | ناجح (pass) | جيد — تحسينات طفيفة ممكنة |
| 500–749 | C | تحذير (warning) | مقبول — مشكلات ملحوظة تحتاج معالجة |
| 200–499 | D | تحذير (warning) | ضعيف — مشكلات كبيرة مكتشفة |
| 0–199 | F | فشل (fail) | حرج — يتطلب اهتمامًا فوريًا |

### حساب الدرجة الإجمالية

الدرجة الإجمالية للهدف هي **معدل موزون** لجميع درجات الفحوصات:

```
الدرجة الإجمالية = مجموع(درجة_الفحص × وزن_الفحص) / مجموع(وزن_الفحص)
```

كل فاحص له **وزن على مستوى الفاحص** (مثلاً SSL = 20.0) وكل فحص ضمن الفاحص له **وزن خاص به**. تُستخدم أوزان الفحوصات في الحساب الإجمالي.

### مستويات الخطورة (Severity)

| الخطورة | المعنى |
|---------|--------|
| `critical` (حرج) | خطر أمني فوري؛ الاستغلال سهل |
| `high` (عالي) | ثغرة كبيرة؛ يجب إصلاحها بشكل عاجل |
| `medium` (متوسط) | مشكلة ملحوظة؛ يجب معالجتها على المدى القريب |
| `low` (منخفض) | مخاوف طفيفة؛ تُصلح عند الإمكان |
| `info` (معلوماتي) | نتيجة معلوماتية؛ لا يلزم إجراء |

### مستويات الثقة (Confidence)

لكل فحص نسبة ثقة تعكس موثوقية الكشف:

| الثقة | المعنى |
|-------|--------|
| 100% | حتمي — الترويسة موجودة/غائبة، تطابق دقيق للإصدار |
| 85–95% | ثقة عالية — مطابقة أنماط موثوقة |
| 70–80% | ثقة متوسطة — كشف استدلالي (Heuristic) |
| 60% | ثقة منخفضة — يعتمد على الشبكة أو مؤشرات غير مباشرة |

## 1. فاحص SSL/TLS

**الفئة:** `ssl` | **الوزن:** 20.0 | **الفحوصات:** 4

يقيّم إعدادات أمان تشفير SSL/TLS (SSL/TLS) للموقع المستهدف، بما في ذلك توفر HTTPS، صلاحية الشهادة، إصدار بروتوكول TLS، وسلوك إعادة التوجيه من HTTP إلى HTTPS.

### 1.1 تمكين HTTPS (HTTPS Enabled)

- **وزن الفحص:** 50
- **ما يفحصه:** هل الموقع متاح عبر HTTPS.
- **آلية العمل:** يرسل طلب HTTP GET إلى `https://target` مع مهلة 10 ثوانٍ (يتم تجاوز التحقق من الشهادة لاختبار الاتصال). يقيس زمن الاتصال.
- **التقييم:**

  | الشرط | الدرجة | الخطورة | الحالة |
  |-------|:------:|---------|--------|
  | HTTPS متاح، زمن الاستجابة > 5000 مللي ثانية | 800 | منخفض | تحذير |
  | HTTPS متاح، زمن الاستجابة <= 5000 مللي ثانية | 1000 | معلوماتي | ناجح |
  | HTTPS غير متاح (خطأ اتصال) | 0 | حرج | فشل |

- **OWASP:** A02:2021 إخفاقات التشفير (Cryptographic Failures)
- **CWE:** CWE-319 نقل النص الواضح للمعلومات الحساسة (Cleartext Transmission)
- **CVSS:** 7.5 (عالي) — `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N`
- **الثقة:** 100%
- **المعالجة:** تمكين HTTPS عن طريق تثبيت شهادة SSL/TLS. شهادات مجانية متاحة من Let's Encrypt. تأكد من إعداد الخادم للاستماع على المنفذ 443.

### 1.2 صلاحية الشهادة (Certificate Validity)

- **وزن الفحص:** 50
- **ما يفحصه:** هل شهادة SSL صالحة، غير منتهية الصلاحية، ليست موقعة ذاتيًا، وكم يومًا متبقيًا حتى انتهاء صلاحيتها.
- **آلية العمل:** ينشئ اتصال TLS على `host:443`، يفحص تواريخ NotBefore/NotAfter للشهادة، المُصدر، الموضوع، أسماء DNS، ويتحقق مما إذا كانت الشهادة موقعة ذاتيًا.
- **التقييم:**

  | الشرط | الدرجة | الخطورة | الحالة |
  |-------|:------:|---------|--------|
  | لا يمكن إنشاء اتصال TLS | 0 | حرج | فشل |
  | لا توجد شهادات | 0 | حرج | فشل |
  | الشهادة منتهية أو لم تبدأ بعد | 0 | حرج | فشل |
  | شهادة موقعة ذاتيًا (Self-signed) | 400 | متوسط | تحذير |
  | صالحة، أكثر من 90 يومًا حتى الانتهاء | 1000 | معلوماتي | ناجح |
  | صالحة، 61–90 يومًا حتى الانتهاء | 900 | معلوماتي | ناجح |
  | صالحة، 31–60 يومًا حتى الانتهاء | 750 | منخفض | ناجح |
  | صالحة، 15–30 يومًا حتى الانتهاء | 500 | متوسط | تحذير |
  | صالحة، 8–14 يومًا حتى الانتهاء | 300 | عالي | تحذير |
  | صالحة، <= 7 أيام حتى الانتهاء | 150 | عالي | تحذير |

- **OWASP:** A02:2021 إخفاقات التشفير
- **CWE:** CWE-295 التحقق غير السليم من الشهادة (Improper Certificate Validation)
- **CVSS:** 5.3 (متوسط)
- **الثقة:** 100%
- **المعالجة:** جدد الشهادات قبل انتهاء صلاحيتها. استبدل الشهادات الموقعة ذاتيًا بشهادات من جهة إصدار معتمدة. استخدم أدوات التجديد التلقائي (مثل certbot لـ Let's Encrypt).

### 1.3 إصدار TLS (TLS Version)

- **وزن الفحص:** 50
- **ما يفحصه:** أي إصدار من بروتوكول TLS يتفاوض عليه الخادم، وهل مجموعة التشفير (Cipher Suite) قوية.
- **آلية العمل:** ينشئ اتصال TLS ويقرأ إصدار البروتوكول ومجموعة التشفير. مجموعات التشفير القوية هي AEAD (AES-GCM, ChaCha20-Poly1305) مع تبادل مفاتيح ECDHE.
- **التقييم:**

  | الشرط | الدرجة | الخطورة | الحالة |
  |-------|:------:|---------|--------|
  | TLS 1.3 | 1000 | معلوماتي | ناجح |
  | TLS 1.2 مع مجموعة تشفير قوية (AEAD+ECDHE) | 850 | معلوماتي | ناجح |
  | TLS 1.2 مع مجموعة تشفير ضعيفة | 700 | منخفض | ناجح |
  | TLS 1.1 (مهمل) | 200 | عالي | تحذير |
  | TLS 1.0 أو أقدم | 0 | حرج | فشل |

- **OWASP:** A02:2021 إخفاقات التشفير
- **CWE:** CWE-326 قوة تشفير غير كافية (Inadequate Encryption Strength)
- **CVSS:** 7.5 (عالي)
- **الثقة:** 100%
- **المعالجة:** اضبط الخادم لدعم TLS 1.2+ فقط. عطّل TLS 1.0 و 1.1. فضّل TLS 1.3. استخدم مجموعات تشفير قوية.

### 1.4 إعادة التوجيه من HTTP إلى HTTPS (HTTP to HTTPS Redirect)

- **وزن الفحص:** 50
- **ما يفحصه:** هل يتم إعادة توجيه طلبات HTTP إلى HTTPS، وما رمز حالة إعادة التوجيه المستخدم.
- **التقييم:**

  | الشرط | الدرجة | الخطورة | الحالة |
  |-------|:------:|---------|--------|
  | إعادة توجيه 301 دائمة إلى HTTPS | 1000 | معلوماتي | ناجح |
  | إعادة توجيه 302 مؤقتة إلى HTTPS | 850 | منخفض | ناجح |
  | إعادة توجيه 3xx أخرى إلى HTTPS | 700 | منخفض | ناجح |
  | لا إعادة توجيه، لكن HTTPS متاح بشكل منفصل | 400 | متوسط | تحذير |
  | HTTP غير قابل للوصول، HTTPS يعمل مباشرة | 400 | متوسط | تحذير |
  | HTTP لا يعيد التوجيه و HTTPS غير متاح | 0 | عالي | فشل |

- **OWASP:** A02:2021 إخفاقات التشفير
- **CWE:** CWE-319 نقل النص الواضح
- **CVSS:** 5.3 (متوسط)
- **الثقة:** 100%
- **المعالجة:** اضبط إعادة توجيه 301 دائمة من HTTP إلى HTTPS. في Apache: `RewriteRule ^(.*)$ https://%{HTTP_HOST}$1 [R=301,L]`. في Nginx: `return 301 https://$host$request_uri;`.

## 2. فاحص ترويسات الأمان

**الفئة:** `headers` | **الوزن:** 20.0 | **الفحوصات:** 7

يقيّم ترويسات استجابة HTTP الأمنية الأساسية التي تحمي من هجمات الويب الشائعة.

### 2.1 ترويسة HSTS (Strict-Transport-Security)

- **وزن الفحص:** 5.0
- **ما يفحصه:** وجود وإعداد ترويسة `Strict-Transport-Security`.
- **التقييم:**

  | الشرط | الدرجة | الخطورة |
  |-------|:------:|---------|
  | الترويسة مفقودة | 0 | حرج |
  | max-age = 0 (معطلة فعليًا) | 100 | — |
  | max-age < 6 أشهر (15768000 ثانية) | 400 | — |
  | max-age >= 6 أشهر و < سنة | 650 | — |
  | max-age >= سنة، بدون includeSubDomains | 800 | — |
  | max-age >= سنة + includeSubDomains، بدون preload | 920 | — |
  | max-age >= سنة + includeSubDomains + preload | 1000 | — |

- **OWASP:** A05:2021 خطأ في الإعدادات الأمنية (Security Misconfiguration)
- **CWE:** CWE-523 نقل بيانات الاعتماد غير المحمي
- **CVSS:** 6.1 (متوسط)
- **الثقة:** 100%
- **المعالجة:** أضف الترويسة: `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`

### 2.2 سياسة أمان المحتوى (Content Security Policy - CSP)

- **وزن الفحص:** 5.0
- **ما يفحصه:** وجود وقوة ترويسة `Content-Security-Policy`.
- **التقييم:**

  | الشرط | الدرجة | الخطورة |
  |-------|:------:|---------|
  | الترويسة مفقودة | 0 | عالي |
  | متساهلة جدًا (`default-src *`) | 200 | — |
  | تحتوي `unsafe-inline` و `unsafe-eval` | 400 | — |
  | تحتوي `unsafe-inline` فقط | 700 | — |
  | مصادر محددة، بدون توجيهات غير آمنة | 1000 | — |

- **OWASP:** A03:2021 الحقن (Injection)
- **CWE:** CWE-79 XSS
- **CVSS:** 6.1 (متوسط)
