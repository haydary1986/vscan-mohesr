  1. في Cloudflare ← DNS ← Records
  2. أضف سجل جديد:

2.2 تقوية سجل SPF 725/1000 +2 نقطة

المشكلة: SPF يستخدم ~all (soft fail) بدلاً من -all (hard fail).

  1. ابحث عن سجل TXT الذي يبدأ بـ v=spf1
  2. غيّر ~all إلى -all
// قبل: v=spf1 include:_spf.google.com ~all // بعد: v=spf1 include:_spf.google.com -all

إعدادات سيرفر Plesk

⏱ 20 دقيقة
3.1 تعطيل HTTP Methods الخطرة 75/1000 +30 نقطة

المشكلة: طرق TRACE, DELETE, PUT, PATCH كلها مفتوحة مما يسمح بهجمات XST وحذف/تعديل الملفات.

الحل — في Plesk:

  1. اذهب إلى Domains ← الموقع ← Apache & nginx Settings
  2. في Additional nginx directives أضف:
# Block dangerous HTTP methods if ($request_method !~ ^(GET|HEAD|POST|OPTIONS)$) { return 444; } # Hide server tokens server_tokens off;

أو — في ملف .htaccess:

# Block dangerous HTTP methods RewriteEngine On RewriteCond %{REQUEST_METHOD} ^(TRACE|DELETE|PUT|PATCH|CONNECT) [NC] RewriteRule .* - [F,L]
3.2 حماية لوحة Admin 125/1000 +7 نقاط

المشكلة: مسار /admin/ متاح للجميع — يسهّل هجمات brute force.

في .htaccess أضف:

# Protect admin - return 404 for non-allowed IPs RewriteEngine On RewriteCond %{REQUEST_URI} ^/(wp-admin|admin) [NC] RewriteCond %{REMOTE_ADDR} !^YOUR.OFFICE.IP.HERE$ RewriteCond %{REQUEST_URI} !^/wp-admin/admin-ajax\.php [NC] RewriteRule .* - [R=404,L]
استبدل YOUR.OFFICE.IP.HERE بعنوان IP مكتبك. يمكنك إضافة أكثر من IP بنسخ سطر RewriteCond.
3.3 إرجاع 404 بدل 403 للملفات الحساسة 725/1000 +3 نقاط

المشكلة: ملفات .env, .git, .htaccess, wp-config.php.bak ترجع خطأ 403 مما يؤكد وجودها.

# Return 404 for sensitive files (hide their existence) RewriteRule ^\.env$ - [R=404,L] RewriteRule ^\.git - [R=404,L] RewriteRule ^\.htaccess$ - [R=404,L] RewriteRule \.bak$ - [R=404,L] RewriteRule \.sql$ - [R=404,L] RewriteRule \.log$ - [R=404,L] RewriteRule ^readme\.html$ - [R=404,L] RewriteRule ^license\.txt$ - [R=404,L]
